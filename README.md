# Azure-And-Cloud-Computing
# ☁️ My Azure and Cloud Computing Notes

## 📌 Overview

This README is my personal summary of **cloud computing** and **Microsoft Azure**.  
It explains what cloud computing is, why businesses use it, and the main Azure services and cloud models I need to understand.

Cloud computing means using computing services over the internet instead of only using local computers or physical servers. These services can include storage, databases, networking, software, analytics, and virtual machines.

---

## 🌍 What Cloud Computing Can Do

Cloud computing can help me and organisations to:

- 💾 Store and access data online
- 🌐 Run websites and applications
- 🗄️ Host databases
- 🔁 Back up files
- 📊 Analyse data
- 📈 Scale resources up or down
- 🏠 Access systems remotely

**Example:**  
A business can store customer data in the cloud and allow staff to access it securely from different locations.

---

## 🏢 Benefits of Cloud Computing

Cloud computing can benefit businesses because they do not need to buy and maintain lots of physical servers.

Main benefits include:

- 💷 Lower hardware costs
- 📈 Easy scaling
- 🌍 Remote access
- 🔄 Backup and disaster recovery
- 🛡️ Better reliability
- ⚡ Faster setup of applications and databases
- 🧰 Access to modern tools

---

## 🖥️ Alternative to Cloud Computing

The main alternative is **on-premises infrastructure**.

This means a company owns and manages its own:

- 🖥️ Servers
- 💾 Storage
- 🌐 Networks
- 🏢 Data centres

This gives more control, but it can cost more because the company must pay for hardware, maintenance, electricity, cooling, and IT staff.

---

## 🌐 Major Cloud Providers

The three major cloud providers are:

### 🔵 Microsoft Azure

Microsoft’s cloud platform for virtual machines, databases, storage, networking, analytics, AI, and security.

### 🟠 Amazon Web Services AWS

A large cloud provider with many services for compute, storage, databases, networking, and machine learning.

### 🟢 Google Cloud Platform GCP

A cloud provider known for data analytics, machine learning, storage, and application hosting.

---

## 🔷 Microsoft Azure

Azure is Microsoft’s cloud computing platform.  
It helps users create, manage, and secure cloud resources.

Useful Azure services include:

- 🖥️ **Azure Virtual Machines**: Cloud-based servers
- 🗄️ **Azure SQL Database**: Managed relational database service
- 📦 **Azure Blob Storage**: Stores files, images, backups, and logs
- 🌐 **Azure App Service**: Hosts websites and APIs
- ⚙️ **Azure Functions**: Runs code without managing servers
- 🔁 **Azure Data Factory**: Moves and transforms data
- 👤 **Microsoft Entra ID**: Manages users and access
- 📊 **Azure Monitor**: Tracks performance, logs, and alerts

---

## 🧱 Cloud Service Models

### 🖥️ IaaS: Infrastructure as a Service

IaaS gives users cloud infrastructure such as virtual machines, storage, and networking.

The user manages:

- Operating system
- Applications
- Data
- Some security settings

**Example:**  
A company moves its database servers to Azure Virtual Machines because it wants control over the operating system and storage.

---

### 🧑‍💻 PaaS: Platform as a Service

PaaS gives developers a managed platform to build, test, and deploy applications.

The cloud provider manages most of the infrastructure, so developers can focus on the application.

**Example:**  
A developer uses Azure App Service to host a website without managing the server.

---

### 📱 SaaS: Software as a Service

SaaS is ready-to-use software delivered over the internet.

The user does not manage the servers, platform, or infrastructure.

**Example:**  
Microsoft 365, Outlook, Teams, OneDrive, Netflix, and Google Docs are examples of SaaS.

---

## ☁️ Cloud Deployment Models

### 🌍 Public Cloud

Public cloud is provided by third-party companies such as Azure, AWS, or Google Cloud.

**Example:**  
A small online shop uses Azure to host its website and store customer orders.

---

### 🔒 Private Cloud

Private cloud is used by one organisation only.  
It gives more control, privacy, and security.

**Example:**  
A hospital uses a private cloud to store patient records safely.

---

### 🔀 Hybrid Cloud

Hybrid cloud combines public cloud and private cloud.

It is useful when a company wants to keep sensitive data private but still use public cloud services.

**Example:**  
A bank keeps customer financial records in a private cloud and uses public cloud for its website.

---

### 👥 Community Cloud

Community cloud is shared by organisations with similar needs or rules.

**Example:**  
Several universities share a cloud platform for research and student services.

---

## 💷 Azure Pricing

Azure normally uses a **pay-as-you-go** pricing model.

Costs can depend on:

- 🧰 Services used
- 🌍 Region
- 💾 Storage amount
- 🖥️ Compute size
- 🔁 Network traffic
- ⏱️ Usage time

The **Azure Pricing Calculator** can be used to estimate costs before creating resources.

Ways to manage costs:

- 📴 Shut down unused virtual machines
- 📏 Choose the correct resource size
- 🚨 Set budgets and alerts
- 🔍 Review resources regularly
- 🗑️ Delete unused resources

---

## 🗄️ Data and Databases in Azure

Azure supports different types of data:

- 📊 **Relational data**: Structured data in tables, such as Azure SQL Database
- 🧾 **Non-relational data**: Flexible data such as documents or key-value data
- 📦 **Object storage**: Files, images, backups, and logs
- 📈 **Analytics data**: Data used for reporting and business intelligence

As a data technician, I may use Azure to:

- Create databases
- Query data using SQL
- Move data between systems
- Store files securely
- Prepare data for reports

---

## 🛡️ Security in the Cloud

Cloud security is shared between the cloud provider and the customer.

Azure protects the physical data centres and many services, but users must still manage access and security settings.

Good security practices include:

- 🔐 Use strong passwords
- ✅ Use multi-factor authentication
- 👤 Give users only the access they need
- 🔒 Encrypt sensitive data
- 🔁 Back up important files
- 📊 Monitor logs and alerts
- 🔄 Keep systems updated
- 📜 Follow data protection rules

---

## ⚠️ Risks of Cloud Computing

Cloud computing has some risks, including:

- 💷 Unexpected costs
- 🔓 Poor security settings
- 🌐 Internet dependency
- 🛑 Cloud service outages
- 🔐 Data privacy concerns
- 🔗 Vendor lock-in
- 🧠 Lack of training or knowledge

These risks can be reduced by planning carefully, monitoring usage, and following security best practices.

---

## 🏢 Example Scenario

A large company wants to move its internal database servers to Azure.

The company:

- Already has IT staff
- Wants full control over operating systems
- Wants control over storage configuration

The best cloud service model would be **IaaS** because Azure Virtual Machines allow the company to manage the operating system, storage, and database setup while using cloud infrastructure.
